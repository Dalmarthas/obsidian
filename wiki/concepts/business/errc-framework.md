# What It Is

The ERRC framework is the Blue Ocean tool for redesigning an offer by deciding what to eliminate, reduce, raise, and create. In the current vault, it functions as a practical way to challenge industry defaults instead of treating all existing features, processes, and assumptions as fixed. Main source: [[wiki/sources/2026-04-13-make-the-competition-irrelevant-blue-ocean-strategy|Make the Competition Irrelevant: Blue Ocean Strategy]].

# Why It Matters

- It turns category-creation talk into a concrete redesign exercise.
- It forces strategy to include subtraction, not only addition.
- It helps teams examine whether they are preserving legacy complexity that buyers do not actually want.

# Evidence In This Vault

- The source defines the four moves directly:
  - eliminate taken-for-granted factors that add little value
  - reduce overbuilt elements below industry norm
  - raise the dimensions that matter more to target buyers
  - create new value dimensions the industry has ignored
- The Yellow Tail case uses the tool to remove wine complexity, shrink choice, increase ease and fun, and create a more approachable category frame.
- The concept gives the branch a stronger bridge between customer insight and offer redesign.

# Source Comparisons

- Compared with [[wiki/concepts/business/framework-driven-action|Framework-Driven Action]], ERRC is narrower and more strategic-product-specific. Framework-driven action is a general execution preference; ERRC is a concrete redesign tool for market moves.
- Compared with [[wiki/concepts/business/jobs-to-be-done|Jobs to Be Done]], ERRC comes later in the process. Jobs identifies the demand-side insight; ERRC translates that insight into specific offer changes.
- Compared with [[wiki/concepts/business/value-innovation|Value Innovation]], ERRC is the working grid used to produce the value innovation outcome.

# Tensions / Open Questions

- Teams can fill the grid mechanically without developing a genuinely better category insight.
- The framework is strong on redesign logic but weaker on validation order, so it should not replace testing.
- The branch still needs more examples from service businesses and software, not only consumer packaged goods.
