# Overview

David Allen enters this vault as the clearest current source on externalizing commitments into a trusted system so attention can return to execution instead of memory maintenance. His role is narrower than the broader self-management branch but highly practical: he gives the vault a formal task-management operating model built around capture, clarification, lists, projects, and review cadence.

# Role In This Vault

- He gives the productivity branch a stronger anti-overwhelm logic that treats stress as partly a systems failure rather than only a prioritization or discipline failure.
- He adds a concrete externalization model for how commitments should move from the head into stable containers.
- He sharpens the difference between vague intentions and defined next actions.
- He adds a stronger weekly-review discipline connecting immediate tasks back to longer-horizon goals and projects.

# Evidence In This Vault

- The current GTD summary argues that uncaptured commitments consume background attention and reduce presence.
- It treats capture, process, and review as a minimum viable maintenance loop for a trusted personal operating system.
- It adds a `next action` rule that makes task definition more concrete than many adjacent productivity branches do.
- It also adds project lists, follow-up lists, context lists, and someday-maybe as structural containers rather than as ad hoc list habits.
