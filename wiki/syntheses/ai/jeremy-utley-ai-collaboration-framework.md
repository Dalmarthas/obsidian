# Question

What collaboration framework does the current vault extract from Jeremy Utley's AI creativity and productivity talk?

# Answer

The current source supports a six-part collaboration framework:

1. Treat the model like an eager intern that needs coaching, not like a perfect oracle.
2. Supply rich context so the task is actually understandable in your real situation.
3. Use role assignment or roleplaying when you need stronger perspective, critique, or simulation.
4. Use examples to anchor tone, structure, or logic more precisely.
5. Ask the model to request missing information before answering when the brief is incomplete.
6. Critique, iterate, and refine instead of treating the first answer as the finished result.

The central shift is from command-writing to collaboration design.

# Basis

- [[wiki/sources/2026-04-09-jeremy-utley-ai-creativity-and-productivity]] frames effective AI use as coaching and collaboration rather than technical wizardry.
- The source emphasizes context engineering, reverse prompting, few-shot examples, role assignment, and iterative critique as the highest-leverage techniques.
- It also makes the practical claim that human imagination and briefing quality often limit results more than model capability does.

# Gaps

- The branch still needs more evidence on which collaboration techniques work best by task type and by model family.
- It also needs stronger comparisons between collaboration-first prompting and more automated workflow architectures.
