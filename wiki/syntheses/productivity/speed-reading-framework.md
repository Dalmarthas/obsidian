# Question

What reusable reading framework emerges from the current Kam Knight speed-reading source?

# Answer

The current source supports a six-part speed-reading framework:

1. Start with a concrete extraction goal.
   Reading improves when the text is approached as a search for specific value rather than as a passive march across every line.

2. Preview the structure before committing attention.
   Use T.H.I.E.V.E.S. to surface the book's likely argument, priorities, and likely payoff before deep reading begins.

3. Reduce fixation count instead of trying to force mental speed.
   The method's central mechanical claim is that reading slows down because the eyes stop too often, not because the mind is incapable of faster intake.

4. Use peripheral capture to read in clusters.
   Shift from direct fixation on every word to gathering groups of words around each fixation point.

5. Push pace only with comprehension checks.
   Higher speed is useful when it preserves or improves understanding; if summary quality collapses, the speed gain is fake.

6. Match the method to the material.
   The protocol appears strongest for moderate-complexity nonfiction and weaker for text that demands symbol-level precision or slow inferential unpacking.

# Comparisons

- Compared with [[wiki/syntheses/productivity/barbara-oakley-learning-framework|Barbara Oakley Learning Framework]], this framework is narrower and more intake-oriented. Oakley explains chunking, recall, and focused-versus-diffuse processing across learning broadly; this framework explains how to enter a reading task with better targeting and less attentional drag.
- Compared with [[wiki/syntheses/productivity/attention-and-focus-framework|Attention and Focus Framework]], this page is more task-specific. The attention synthesis explains how to create the conditions for concentration; this framework explains how to use those conditions to move through text more efficiently.
- Compared with [[wiki/concepts/productivity/learning-pyramid|Learning Pyramid]], this framework mostly improves the lower levels of the ladder. It makes understanding more efficient, but still requires later recall, explanation, and application to verify that learning actually stuck.

# Gaps

- The source is still a compact secondary summary rather than a strong evidence review.
- The vault needs a counter-source from reading research or a skeptical treatment of speed reading so the branch does not overgeneralize one operational method.
- The current source is strongest on reading efficiency and weaker on long-term retention, transfer, and genre-specific limits.
