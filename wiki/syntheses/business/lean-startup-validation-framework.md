# Question

What reusable validation framework does the Lean Startup source add to the vault?

# Answer

The current source base supports a six-step model:

1. Start with the vision, but isolate the riskiest assumption.
   Do not try to validate the whole dream at once. Identify the belief that would most damage the business if it proved false.

2. Build the smallest credible test for that assumption.
   The MVP is not a ceremonial mini-product. It is the cheapest serious instrument that can expose whether the assumption survives contact with reality.

3. Prefer behavior over opinion.
   Purchases, pre-orders, sign-ups, or other commitments matter more than compliments, interviews, or social encouragement.

4. Use low-infrastructure testing modes when possible.
   Concierge MVPs test value manually. Smokescreen MVPs test demand before full buildout.

5. Read the result as a decision signal, not as ego feedback.
   The point is not whether people "liked" the idea. The point is whether the observed behavior supports continuing on the same hypothesis.

6. Pivot or persevere, then repeat.
   Learning is cyclical. Each round should either strengthen confidence in the direction or force a meaningful change in approach.

# Comparisons

- Compared with [[wiki/syntheses/business/freymvork-vybora-nishi-dlya-biznesa|Фреймворк выбора ниши для бизнеса]], this framework is more universal and less tied to niche selection, segmentation, KEF, or Russian-market channel tactics.
- The Russian niche framework is stronger as an execution manual for choosing where to enter and how to test early demand inside a specific market-selection workflow.
- The Lean Startup framework is stronger as a general operating principle for uncertainty: identify the assumption, test behavior, and use the result to pivot or persevere.
- Together they now give the business branch both layers:
  - a broad logic for learning under uncertainty
  - a more specific niche-entry playbook for rapid commercial validation

# Gaps

- This framework currently rests on one secondary summary rather than a broader Eric Ries branch.
- The vault still needs stronger counterexamples from markets where lightweight MVP tests are less feasible.
- The current source mentions the build-measure-learn cycle clearly, but it does not deepen innovation accounting or the wider Lean Startup organizational layer.
